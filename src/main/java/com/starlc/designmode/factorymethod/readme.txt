工厂方法：
	提供一个方法接口，具体的方法由实现类去实现
	eg. 计算器  createOpreator(); 由实现类根据实际需要实现此方法，并返回对应的操作类
	
	
对比简单工厂 和 工厂方法：
	1、简单工厂模式最大的优点在于工厂类中包含了必要的逻辑判断，根据客户端的选择条件动态实例化相关的类，对于客户端来说，去除了与具体产品的依赖。但是当新增功能时需要修改工厂方法。（开放-封闭原则）
	2、工厂方法定义一个用于创建对象的接口，让子类决定具体去实例化哪一个类。
	
	为了避免分支节点判断太多，后续可以考虑用反射来进行实现
	
	工厂方法的缺点就是需要为不同类型的实现 创建太多子类工厂
